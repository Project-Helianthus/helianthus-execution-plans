## Objective

Provide a single, gateway-owned, JSON-formatted, atomically-written runtime
state file under `/data/runtime_state.json` consolidating:

1. **Canonical Helianthus instance identity** (`meta.instance_guid`) — the
   uuid4 stable identity for HA `config_entry.unique_id` rebind across
   restarts. Generated by the add-on (locked rediscovery contract);
   persisted by the gateway (sole writer per AD07).
2. **Cached SourceAddressSelection result** (`ebus.self`) — a HINT for subsequent
   SourceAddressSelection sessions. Carries `last_admitted_source`, `last_admitted_at`,
   `selection_method`, `companion_target`. Never bypasses SourceAddressSelector validation
   (locked startup-admission contract preserved).
3. **Cached observed bus members** (`ebus.known_bus_members[]`) — for
   startup acceleration via bounded directed `07 04` re-validation. Cache
   is wiped per-startup of any non-responder; verified responders are
   refreshed.

  SourceAddressSelector always validates).
- Gateway: post-warmup directed `07 04` revalidation via
  `helianthus-ebusreg.ScanDirected`, capped at 32 probes per cycle,
  ordered by `last_seen_at` DESC.
- HA add-on: AD09b read precedence; AD09a deploy-error guardrail; AD27
  `-instance-guid-source` flag (compat-first).
- Atomic + crash-safe writes (single serialized writer; temp in target
  dir; fsync(temp) required; fsync(parent_dir) best-effort; EXDEV =
  write failure preserving old file).
- Transport-gate: 88-case dry-run baseline at M0A and re-verify at M8.

  deferred to v2 (Codex R5 A2: namespace separation).
- Other plugin namespaces (`gree.*`, `eebus.*`,
  `ebus.proxy_clients[]`) — out of v1 scope.
- Register-value caching, Prometheus-metric persistence — explicitly
  rejected by operator after debate.

## Plugin-namespace boundary

The `ebus.*` namespace is owned by the gateway and reflects gateway-side
observation persistence. v2+ will introduce other plugin namespaces:

- `gree.*` — Gree HVAC adapter cache (planned).
- `eebus.*` — eEBus adapter cache (planned).
- `ebus.proxy_clients[]` — proxy adapter client persistence (planned).

The schema versioning is **two-tier**:

- Top-level `schema_version` int — file format compatibility.
- Per-plugin `<plugin>.schema_version` int — per-plugin schema
  compatibility.

Plugin namespace mismatch (e.g., `ebus.schema_version=99` on a gateway
that only knows v1) is **silently ignored** (the namespace is dropped
from the in-memory load), never crashes startup. Other namespaces still
load normally. This is N3 in the falsifiability gate.

## Sole-writer model

The gateway is the sole writer of `/data/runtime_state.json`. The add-on
**reads** the file at startup to extract `meta.instance_guid` (per AD09b
precedence), then passes the chosen identity via `-instance-guid` flag
(existing locked rediscovery contract). The add-on never writes the
file post-v1.

This eliminates cross-process write contention. The add-on read-vs-gateway-
write contention window during startup (gateway eagerly persists in first
second; add-on may already have read) is closed by AD26: add-on tolerates
ENOENT on open() with bounded retries (3 attempts, 100ms backoff).

The add-on **still owns instance_guid generation** per the locked
rediscovery plan invariant. Generation happens in case (3) of AD09b ("both
files absent → generate fresh uuid4"). The generated value is passed via
`-instance-guid` flag; the gateway eagerly persists it to runtime_state.json
within the first second of startup.
