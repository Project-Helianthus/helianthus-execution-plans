Each row is one future executable issue owned by exactly one repository. The existing
`.github` governance issue creates the four planned repositories; destination bootstrap
rows depend on it and run only after each repository exists. This draft creates none.
This map contains 43 issues. Milestones are groupings only.

Issue completion never substitutes for either GO. `NO_GO` and `STOP` preserve evidence but
are not progress.
The M3 Fronius and M7 Growatt dispositions are not conditional gates: either completed M3
disposition releases M4, and completed Growatt `NO_ADMISSIBLE_PROFILE` releases Huawei
while preserving the pre-published public evidence/unsupported disposition and creating no
code, catalog entry, support claim, or later companion docs change.
FMV3-M1-00 is the one public companion docs issue/PR for both M1 runtime and the complete M2
contract set; no additional M2 docs issue is created.
The docs lane remains serialized: FMV3-M5-02 is the earlier canonical PV issue, FMV3-M5-04
then implements/tests candidate semantic MCP, FMV3-M5-03 locks that exact version, and
FMV3-M5-09 is the one later PUBLIC_GRAPHQL_M2M_V1 companion issue before FMV3-M5-05.
FMV3-M7-01 follows M5-09 in that docs lane, publishes the complete
Growatt candidate/admission contract before closing, and is companion for M7-02..04. The
same public repository owns M6-00 and M8-00 pre-implementation binding companions plus
M6-03 sanitized post-lab knowledge publication; its repository mutex prevents concurrent
issues/PRs while preserving independent M6 and M8 dependency branches. The
topology/preflight repository mutex permits at most one active issue and one
active PR per repository.
